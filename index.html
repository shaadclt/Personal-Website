Exploring Natural Language Processing in Python: A Comprehensive Guide



Introduction

In the realm of data science and machine learning, Natural Language Processing (NLP) plays a pivotal role in extracting valuable insights from textual data. In this blog post, we will embark on a journey through various NLP techniques using Python, leveraging popular libraries such as spaCy, TextBlob, and scikit-learn. Our focus will be on visualizing, analyzing sentiment, performing Named Entity Recognition (NER), and diving into the world of Topic Modeling.

1. Importing Libraries

To begin our NLP journey, we import essential libraries for data manipulation, visualization, and natural language processing. These libraries include Pandas for data handling, Plotly Express for interactive visualizations, WordCloud for creating word clouds, Matplotlib for additional plotting capabilities, TextBlob for sentiment analysis, spaCy for Named Entity Recognition (NER), and scikit-learn for Topic Modeling. 

        import pandas as pd
        import plotly.express as px
        from wordcloud import WordCloud
        import matplotlib.pyplot as plt
        from textblob import TextBlob
        import spacy
        from collections import defaultdict
        from collections import Counter
        from sklearn.feature_extraction.text import CountVectorizer
        from sklearn.decomposition import LatentDirichletAllocation

2. Loading Data

In this step, we load our dataset using Pandas. The dataset, stored in a CSV file named "articles.csv," contains information that we will analyze using various NLP techniques. 

        nlp = spacy.load('en_core_web_sm')
        data = pd.read_csv("articles.csv", encoding='latin-1')
        data.head()

3. Visualizing the Word Cloud

Word clouds offer a visually appealing representation of the most frequent words in a collection of text. Here, we generate a word cloud based on the titles of the articles using the WordCloud library and visualize it with Plotly Express. 

        titles_text = ' '.join(data['Title'])
        wordcloud = WordCloud(width=800, height=400, background_color='white').generate(titles_text)
        fig = px.imshow(wordcloud, title='Word Cloud of Titles')
        fig.update_layout(showlegend=False)
        fig.show()

4. Analyzing the Distribution of Sentiments

Sentiment analysis helps us understand the emotional tone of the articles. Using TextBlob, we calculate the sentiment polarity for each article and visualize the distribution using a histogram with Plotly Express. 

        data['Sentiment'] = data['Article'].apply(lambda x: TextBlob(x).sentiment.polarity)
        fig = px.histogram(data, x='Sentiment', title='Sentiment Distribution')
        fig.show()

5. Performing Named Entity Recognition

Named Entity Recognition (NER) involves identifying entities such as people, organizations, and locations within the text. Using spaCy, we extract named entities from the articles and visualize the top entities with a bar chart. 

        def extract_named_entities(text):
            doc = nlp(text)
            entities = defaultdict(list)
            for ent in doc.ents:
                entities[ent.label_].append(ent.text)
            return dict(entities)
        
        data['Named_Entities'] = data['Article'].apply(extract_named_entities)
        
        entity_counts = Counter(entity for entities in data['Named_Entities'] for entity in entities)
        entity_df = pd.DataFrame.from_dict(entity_counts, orient='index').reset_index()
        entity_df.columns = ['Entity', 'Count']
        
        fig = px.bar(entity_df.head(10), x='Entity', y='Count', title='Top 10 Named Entities')
        fig.show()

6. Topic Modeling

Topic Modeling is a technique to discover topics present in a collection of text. In this step, we use scikit-learn to perform Latent Dirichlet Allocation (LDA), a popular topic modeling method. We visualize the distribution of topics with a bar chart. 

        vectorizer = CountVectorizer(max_df=0.95, min_df=2, max_features=1000, stop_words='english')
        tf = vectorizer.fit_transform(data['Article'])
        lda_model = LatentDirichletAllocation(n_components=5, random_state=42)
        lda_topic_matrix = lda_model.fit_transform(tf)
        
        topic_names = ["Topic " + str(i) for i in range(lda_model.n_components)]
        data['Dominant_Topic'] = [topic_names[i] for i in lda_topic_matrix.argmax(axis=1)]
        topic_counts = data['Dominant_Topic'].value_counts().reset_index()
        topic_counts.columns = ['index', 'count']
        
        fig = px.bar(topic_counts, x='index', y='count', title='Topic Distribution')
        fig.show()

Conclusion

This comprehensive guide has walked you through the essential steps of natural language processing using Python. From creating captivating word clouds and analyzing sentiment distributions to extracting named entities and uncovering hidden topics, you now possess the tools to extract valuable insights from textual data. Experiment, explore, and unlock the full potential of NLP in your data science projects. Happy coding!

Building a Next Word Prediction Model using LSTM and TensorFlow



Introduction

Predicting the next word in a sequence is a fascinating application of natural language processing (NLP). In this blog post, we'll walk through the process of building a Next Word Prediction (NWP) model using the Long Short-Term Memory (LSTM) architecture with TensorFlow. We'll cover tokenization, creating input-output pairs, building a neural network, training the model, and generating predictions.

1. Importing Libraries

We start by importing the necessary libraries for our project, including NumPy for numerical operations and TensorFlow for building and training neural networks. We specifically import components from TensorFlow's Keras module, which provides high-level building blocks for creating neural networks. 

              import numpy as np
              import tensorflow as tf
              from tensorflow.keras.preprocessing.text import Tokenizer
              from tensorflow.keras.preprocessing.sequence import pad_sequences
              from tensorflow.keras.models import Sequential
              from tensorflow.keras.layers import Embedding, LSTM, Dense

2. Loading Data

The dataset used for this project is the text from Sherlock Holmes stories, loaded from a plain text file. This text will be the basis for training our NLP model. 

              with open('sherlock-holm.es_stories_plain-text_advs.txt', 'r', encoding='utf-8') as file:
                text = file.read()

3. Tokenizing the Text

Tokenization is the process of breaking down text into words or subwords. We use the Tokenizer class from Keras to tokenize the text and create a sequence of words. The total number of unique words in the text is calculated as total_words. 

              tokenizer = Tokenizer()
              tokenizer.fit_on_texts([text])
              total_words = len(tokenizer.word_index) + 1

4. Creating Input-Output Pairs

We create input-output pairs by sliding a window over the tokenized sequences. For each window, the input is the sequence of words up to a certain point, and the output is the next word in the sequence. 

              input_sequences = []
              for line in text.split('\n'):
                  token_list = tokenizer.texts_to_sequences([line])[0]
                  for i in range(1,len(token_list)):
                      n_gram_sequence = token_list[:i+1]
                      input_sequences.append(n_gram_sequence)

5. Padding Input Sequences

To train our model efficiently, we pad the input sequences to have equal length. The maximum sequence length is determined by the longest sequence in our input data. 

              max_sequence_len = max([len(seq) for seq in input_sequences])
              input_sequences = np.array(pad_sequences(input_sequences,maxlen=max_sequence_len, padding='pre'))

6. Splitting Sequences into Input and Output

We separate the input sequences into features (X) and labels (y), where X contains all words except the last one, and y contains the last word in each sequence. 

              X = input_sequences[:,:-1]
              y = input_sequences[:,-1]

7. Converting the Output to One-Hot Encoded Vectors

The output labels (y) are converted to one-hot encoded vectors. This step is necessary for training a categorical classification model. 

              y = np.array(tf.keras.utils.to_categorical(y,num_classes=total_words))

8. Building Neural Network Architecture

We create a sequential model using Keras. The model consists of an embedding layer, an LSTM layer, and a dense layer with a softmax activation function. The embedding layer turns positive integers into dense vectors, and the LSTM layer helps the model understand the context of the input sequences. 

              model = Sequential()
              model.add(Embedding(total_words,100,input_length = max_sequence_len-1))
              model.add(LSTM(150))
              model.add(Dense(total_words,activation='softmax'))
              print(model.summary())

9. Compile and Train the Model

The model is compiled using categorical crossentropy as the loss function and the Adam optimizer. We train the model on our input sequences and labels for 100 epochs. 

              model.compile(loss='categorical_crossentropy',optimizer='adam',metrics=['accuracy'])
              model.fit(X,y,epochs=100,verbose=1)

10. Saving the Model

Once the training is complete, we save the model to a file (nwp_model.h5) for later use or deployment. 

              model.save('nwp_model.h5')

11. Generating Next Word Predictions

We load the saved model and demonstrate how to generate next word predictions given a seed text. The model predicts the next word iteratively based on the input seed text. 

              from tensorflow.keras.models import load_model
              # Load the saved model
              model = load_model('nwp_model.h5')  # Replace 'my_model.h5' with the path to your saved model

              seed_text = "I am excited to"
              next_words = 2

              for _ in range(next_words):
                  token_list = tokenizer.texts_to_sequences([seed_text])[0]
                  token_list = pad_sequences([token_list], maxlen=max_sequence_len - 1, padding='pre')
                  predicted = np.argmax(model.predict(token_list), axis=-1)
                  output_word = ""
                  for word, index in tokenizer.word_index.items():
                      if index == predicted:
                          output_word = word
                          break
                  seed_text += " " + output_word

              print(seed_text)

Conclusion

Building a Next Word Prediction model involves several key steps, from data preparation to model training and prediction generation. The LSTM architecture, known for its ability to capture sequential patterns, proves to be effective in understanding and predicting the next word in a sequence of text. This project provides a foundational understanding of text generation using neural networks. Feel free to modify and expand on this draft to fit the style and requirements of your blog!

Navigating Currency Markets: A Comprehensive Guide to Exchange Rate Forecasting



Introduction

Currency exchange rates play a pivotal role in the global economy, influencing trade, investment, and financial decisions. In this blog post, we embark on a journey of currency exchange rate forecasting using Python. We will leverage data analysis, visualization, and time series forecasting techniques to gain insights into the USD – INR exchange rates.

Importing Libraries and Loading Data:

To kickstart our analysis, we begin by importing essential Python libraries such as pandas, numpy, matplotlib, seaborn, and plotly. We load the dataset containing the USD – INR exchange rates and perform an initial check for missing values. 

              import pandas as pd
              import numpy as np
              import matplotlib.pyplot as plt
              import seaborn as sns
              import plotly.express as px
              import warnings
              warnings.filterwarnings('ignore')

              data = pd.read_csv('INR-USD.csv')
              data.head()

Descriptive Statistics and Visualization:

After ensuring data integrity, we delve into a descriptive analysis of the dataset. Utilizing Plotly Express, we create a line chart to visualize the trend of USD – INR conversion rates over the years. 

              data.isnull().sum()

              data = data.dropna()

              data.describe()

Yearly and Monthly Growth Analysis:

Adding a temporal dimension to our analysis, we explore the aggregated yearly and monthly growth of the conversion rates. Bar charts vividly illustrate the fluctuations, highlighting interesting patterns such as the USD's consistent strength in the second quarter. 

              figure = px.line(data, x="Date", 
                 y="Close", 
                 title='USD - INR Conversion Rate over the years')
              figure.show()

              data["Date"] = pd.to_datetime(data["Date"], format = '%Y-%m-%d')
              data['Year'] = data['Date'].dt.year
              data["Month"] = data["Date"].dt.month
              data.head()

              import plotly.graph_objs as go
              import plotly.io as pio

              # Calculate yearly growth
              growth = data.groupby('Year').agg({'Close': lambda x: (x.iloc[-1]-x.iloc[0])/x.iloc[0]*100})

              fig = go.Figure()
              fig.add_trace(go.Bar(x=growth.index,
                                  y=growth['Close'],
                                  name='Yearly Growth'))

              fig.update_layout(title="Yearly Growth of USD - INR Conversion Rate",
                                xaxis_title="Year",
                                yaxis_title="Growth (%)",
                                width=900,
                                height=600)

              pio.show(fig)


              # Calculate monthly growth
              data['Growth'] = data.groupby(['Year', 'Month'])['Close'].transform(lambda x: (x.iloc[-1] - x.iloc[0]) / x.iloc[0] * 100)

              # Group data by Month and calculate average growth
              grouped_data = data.groupby('Month').mean().reset_index()

              fig = go.Figure()

              fig.add_trace(go.Bar(
                  x=grouped_data['Month'],
                  y=grouped_data['Growth'],
                  marker_color=grouped_data['Growth'],
                  hovertemplate='Month: %{x}
Average Growth: %{y:.2f}%'
              ))

              fig.update_layout(
                  title="Aggregated Monthly Growth of USD - INR Conversion Rate",
                  xaxis_title="Month",
                  yaxis_title="Average Growth (%)",
                  width=900,
                  height=600
              )

              pio.show(fig)

4. Analyzing the Distribution of Sentiments

Sentiment analysis helps us understand the emotional tone of the articles. Using TextBlob, we calculate the sentiment polarity for each article and visualize the distribution using a histogram with Plotly Express. 

        data['Sentiment'] = data['Article'].apply(lambda x: TextBlob(x).sentiment.polarity)
        fig = px.histogram(data, x='Sentiment', title='Sentiment Distribution')
        fig.show()

5. Performing Named Entity Recognition

Named Entity Recognition (NER) involves identifying entities such as people, organizations, and locations within the text. Using spaCy, we extract named entities from the articles and visualize the top entities with a bar chart. 

        def extract_named_entities(text):
            doc = nlp(text)
            entities = defaultdict(list)
            for ent in doc.ents:
                entities[ent.label_].append(ent.text)
            return dict(entities)
        
        data['Named_Entities'] = data['Article'].apply(extract_named_entities)
        
        entity_counts = Counter(entity for entities in data['Named_Entities'] for entity in entities)
        entity_df = pd.DataFrame.from_dict(entity_counts, orient='index').reset_index()
        entity_df.columns = ['Entity', 'Count']
        
        fig = px.bar(entity_df.head(10), x='Entity', y='Count', title='Top 10 Named Entities')
        fig.show()

Forecasting Exchange Rates with SARIMA:

Transitioning into the realm of time series forecasting, we recognize a seasonal pattern in the data using seasonal decomposition. Subsequently, we employ the pmdarima library to determine the optimal parameters (p, d, q) for our SARIMA model. 

              from statsmodels.tsa.seasonal import seasonal_decompose
              result = seasonal_decompose(data["Close"], model='multiplicative', period=24)
              fig = plt.figure()  
              fig = result.plot()  
              fig.set_size_inches(8, 6)
              fig.show()

              from pmdarima.arima import auto_arima
              model = auto_arima(data['Close'], seasonal=True, m=52, suppress_warnings=True)
              print(model.order)

              p, d, q = 2, 1, 0

              from statsmodels.tsa.statespace.sarimax import SARIMAX
              model = SARIMAX(data["Close"], order=(p, d, q), 
                              seasonal_order=(p, d, q, 52))  
              fitted = model.fit() 
              print(fitted.summary())

              predictions = fitted.predict(len(data), len(data)+60)
              print(predictions)

Visualizing Forecasted Results:

In the final stretch, we visualize the forecasted exchange rates alongside the training data. This Plotly figure showcases the model's predictions, providing a tangible representation of the forecasted currency values. 

              # Create figure
              fig = go.Figure()

              # Add training data line plot
              fig.add_trace(go.Scatter(
                  x=data.index,
                  y=data['Close'],
                  mode='lines',
                  name='Training Data',
                  line=dict(color='blue')
              ))

              # Add predictions line plot
              fig.add_trace(go.Scatter(
                  x=predictions.index,
                  y=predictions,
                  mode='lines',
                  name='Predictions',
                  line=dict(color='green')
              ))

              fig.update_layout(
                  title="INR Rate - Training Data and Predictions",
                  xaxis_title="Date",
                  yaxis_title="Close",
                  legend_title="Data",
                  width=900,
                  height=600
              )

              pio.show(fig)
